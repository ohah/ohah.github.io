# 3-2. 코어 개발하기

## 핵심 파싱 구현 (2025-11-23 ~ 2025-11-27)

약 5일 동안 약 130개의 커밋으로 핵심 기능을 구현했다.

## DocInfo 파싱 (2025-11-23 ~ 2025-11-24)

- **DocumentProperties**: 문서 속성 정보
- **IdMappings**: ID 매핑 테이블
- **FaceName**: 글꼴 정보
- **BorderFill**: 테두리 및 채우기 정보
- **CharShape**: 문자 모양 정보
- **TabDef**: 탭 정의
- **Numbering**: 번호 매기기
- **Bullet**: 글머리표
- **Style**: 스타일 정보

## BodyText 파싱 (2025-11-24 ~ 2025-11-25)

문서 본문의 핵심 구조를 파싱했다.

- **BodyText 섹션 파싱**: CFB 경로를 통한 모듈화된 접근
- **ParaCharShape**: 문단별 문자 모양
- **ParaLineSeg**: 문단별 줄 분할 정보
- **ParaRangeTag**: 문단 범위 태그
- **CtrlHeader**: 컨트롤 헤더 파싱
- **ListHeader**: 목록 헤더 파싱

## 컨트롤 및 객체 파싱 (2025-11-25 ~ 2025-11-26)

다양한 컨트롤과 객체를 파싱했다.

- **PAGE_DEF**: 페이지 정의
- **FOOTNOTE_SHAPE**: 각주/미주 모양
- **PAGE_BORDER_FILL**: 페이지 테두리/채우기
- **SHAPE_COMPONENT**: 도형 컴포넌트 (선, 사각형, 타원, 다각형 등)
- **HWPTAG_TABLE**: 표 파싱
- **BinData**: 바이너리 데이터 (이미지 등)
- **PreviewText**: 미리보기 텍스트
- **PreviewImage**: 미리보기 이미지 (base64 인코딩)
- **Scripts**: 스크립트 스토리지
- **XMLTemplate**: XML 템플릿

## 마크다운 변환 및 플랫폼 지원 (2025-11-26 ~ 2025-11-27)

- **마크다운 변환 기능**: HWP를 마크다운으로 변환
- **이미지 렌더링**: 마크다운 출력에 이미지 포함
- **컨트롤 문자 파싱**: 인라인 컨트롤 파라미터를 구조화된 데이터로 파싱
- **React Native 지원**: Craby를 통한 React Native 바인딩 추가
- **예제 프로젝트**: React Native 예제 추가

## 기능 확장 및 개선 (2025-12-01 ~ 현재)

### 최근 주요 기능

- **Header/Footer 파싱**: 머리말/꼬리말 구현
- **SummaryInformation 파싱**: 문서 요약 정보 (VT_LPWSTR 지원)
- **ControlMask 구조**: 비트 플래그를 사용한 성능 최적화
- **CLI 도구**: HWP 파일 변환을 위한 커맨드라인 도구 추가
- **성능 최적화**: 패키지 크기 205MB → 131MB로 최적화

### 코드 품질 개선

- **모듈 구조 개선**: 마크다운 뷰어 모듈을 문서 구조에 맞게 재구성
- **스냅샷 테스트 업데이트**: 모든 변경사항에 대한 테스트 유지
- **문서화 개선**: API 문서, 플랫폼별 예제 추가

## 개발 통계

- **총 커밋 수**: 약 200개 이상
- **주요 개발 기간**: 2025-11-22 ~ 2025-12-06 (약 2주)
- **가장 집중된 기간**: 2025-11-23 ~ 2025-11-27 (5일간 약 130개 커밋)
- **지원 플랫폼**: Node.js, Web, React Native

## 개발 철학

커밋 메시지를 보면 다음과 같은 개발 철학이 보인다:

1. **점진적 구현**: 작은 단위로 나누어 꾸준히 커밋
2. **테스트 주도**: 모든 기능에 대한 스냅샷 테스트 작성
3. **문서화 병행**: 코드와 함께 문서도 함께 업데이트
4. **리팩토링 지속**: 코드 구조를 지속적으로 개선

이러한 접근 방식으로 짧은 기간에 안정적인 HWP 파서를 구현할 수 있었다.
