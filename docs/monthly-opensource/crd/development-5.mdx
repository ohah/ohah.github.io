# 3-5. 터보모듈

터보모듈(TurboModule)은 React Native 아키텍처의 성능을 획기적으로 개선하면서도 기존 브리지 구성을 재설계한 구조적 변화의 핵심 개념입니다. Chrome Remote Debugger(CRD)는 이러한 터보모듈과의 연동을 통해 더욱 강력한 개발자 경험을 제공합니다.

## 터보모듈 기반 아키텍처 개편

터보모듈은 기존 브리지의 단점을 보완하고, 네이티브와 자바스크립트 영역 간 통신 속도를 개선하기 위한 솔루션입니다. CRD는 이에 맞춰 다음의 주요 변경 사항을 적용하였습니다:

1. **JSI(JavaScript Interface) 활용:**
  - 기존 React Native가 사용하는 브리지 대신 JSI를 통해 네이티브 함수를 호출.
  - 성능 병목 해소 및 호출 오버헤드 감소.
2. **페이로드 최적화:**
  - 브리지 통신의 데이터 페이로드를 최적화하여 대규모 데이터 전송 속도 개선.
3. **플러그인 지원 확장:**
  - 플러그인 구조를 터보모듈 틀에 맞춰 재작성, 확장과 유지보수 용이성 증가.

이와 같은 변경 사항은 React Native에서 CRD를 사용할 때 네이티브 모듈 디버깅 속도에서 평균 30% 이상의 성능 향상을 가져왔습니다.

## 주요 기능

### 1. 네이티브와의 동기화 속도 개선

터보모듈에서 CRD는 네이티브 코드 호출 및 데이터 로드 시 다음 기능을 제공합니다:
- 네이티브 모듈 로깅: 브라우저 콘솔에서 실시간.
- 통신 최적화: 네트워크 로그 단순화 및 디버깅.
