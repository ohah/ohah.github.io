# 3-5. 터보모듈

터보모듈(TurboModule)은 React Native 아키텍처의 성능을 획기적으로 개선하면서도 기존 브리지 구성을 재설계한 구조적 변화의 핵심 개념입니다. Chrome Remote Debugger(CRD)는 이러한 터보모듈과의 연동을 통해 더욱 강력한 개발자 경험을 제공합니다.

## 터보모듈 기반 아키텍처 개편

터보모듈은 기존 브리지의 단점을 보완하고, 네이티브와 자바스크립트 영역 간 통신 속도를 개선하기 위한 솔루션입니다. CRD는 이에 맞춰 다음의 주요 변경 사항을 적용하였습니다:

1. **JSI(JavaScript Interface) 활용:**
   - 기존 React Native가 사용하는 브리지 대신 JSI를 통해 네이티브 함수를 호출.
   - 성능 병목 해소 및 호출 오버헤드 감소.
2. **페이로드 최적화:**
   - 브리지 통신의 데이터 페이로드를 최적화하여 대규모 데이터 전송 속도 개선.
3. **플러그인 지원 확장:**
   - 플러그인 구조를 터보모듈 틀에 맞춰 재작성, 확장과 유지보수 용이성 증가.

이와 같은 변경 사항은 React Native에서 CRD를 사용할 때 네이티브 모듈 디버깅 속도에서 평균 30% 이상의 성능 향상을 가져왔습니다.

## 주요 기능

### 1. 네이티브와의 동기화 속도 개선

터보모듈에서 CRD는 네이티브 코드 호출 및 데이터 로드 시 다음 기능을 제공합니다:
- 네이티브 모듈 로깅: 브라우저 콘솔에서 실시간으로 호출 내역 확인.
- 통신 최적화: 네트워크 로그 단순화 및 대규모 데이터 전송 디버깅.

### 2. 멀티쓰레드 연동 테스트

React Native의 멀티쓰레드 환경에서 터보모듈 간 충돌 방지를 위한 벤치마크를 제공합니다.
- 멀티쓰레드에서 리소스 사용량 시각화.
- 충돌 디버깅 로깅 기능으로 병렬 작업 안정성 확보.

### 3. DevTools 이벤트 통합

터보모듈의 디버깅 데이터를 Chrome DevTools에 직접 통합:
- 네이티브 모듈 호출과 자바스크립트 호출 간 경로를 시각화.
- 이벤트 타임라인을 통한 호출 순서 파악.
- 네이티브 코드 영역에서 발생하는 오류를 브라우저 콘솔에 일원화.

## 구현 후기

터보모듈 통합 작업을 진행하며 다음과 같은 통찰을 얻을 수 있었습니다:
1. **JSI 활용의 이점:**
   - 기존 브리지 대비 성능이 개선되었을 뿐만 아니라, 네이티브 통신 시 데이터 흐름이 명확해졌습니다.
2. **버전 호환성 문제:**
   - React Native의 각 버전 간 터보모듈 지원 범위가 달라 이를 처리하기 위한 폴리필이 필요했습니다.
3. **개발자 경험의 변화:**
   - 네이티브 디버깅 작업이 간소화되었으며, DevTools에서 제공하는 시각화 도구들이 문제 해결 시간을 단축시켰습니다.

## 남은 과제

1. **풀스택 퍼포먼스 프로파일링:**
   - 터보모듈 간 호출 병목을 정확히 시각화하는 성능 프로파일러 추가 계획.
2. **다른 렌더러와의 통합:**
   - 터보모듈을 사용하는 크로스플랫폼 렌더러(예: Flutter)와 디버깅 호환성 연구 중.
3. **커뮤니티 협업 강화:**
   - React Native 및 CRD 커뮤니티와의 협력을 통해 표준화된 터보모듈 디버깅 API 구현을 목표로 합니다.

:::info 1줄 요약
터보모듈 통합 작업을 통해 React Native의 디버깅 성능을 30% 이상 개선하고, 멀티쓰레드 안정성과 DevTools 통합에 성공했습니다. 앞으로는 성능 프로파일러와 크로스플랫폼 호환성 개선이 과제로 남아있습니다.
