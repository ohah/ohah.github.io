# 3-4. React Native

React Native의 DevTools 통합은 다른 플랫폼보다도 더 복잡한 도전 과제였지만, 동시에 가장 큰 기회를 제공하는 작업이었다. 기존의 CDP 기반 구조를 활용하면서도 React Native 특유의 환경에 맞는 최적화를 병행해야 했다.

## DevTools 구조 확장

React Native는 웹이 아니기 때문에, 기존 웹 브라우저 기반 구조를 바로 사용할 수 없었다. 이를 위해 기존 DevTools의 구조를 다음과 같이 수정했다:

1. **플랫폼 간 추상화 계층 추가:**
   - React Native는 브라우저 대신 JS 엔진(예: Hermes, JSC) 위에서 실행된다.
   - 이로 인해 CDP를 호출하기 위한 브리지 계층이 필요했다.
2. **네트워크 모듈 재작성:**
   - React Native의 네트워크 계층은 웹 브라우저와 다르기 때문에, WebSocket 연결을 관리하는 네이티브 모듈을 별도로 구현했다.
3. **UI 커스터마이징:**
   - 터치 기반 디버깅과 같은 모바일 특화 기능을 추가하여 사용자 경험을 개선했다.

결과적으로, 기존 DevTools를 크게 재구성하지 않으면서 React Native에서도 동일한 DX(개발자 경험)를 제공할 수 있게 되었다.

## React Native 전용 기능

React Native 환경에 맞는 특화된 기능도 추가했다:

### Hermes 디버깅 통합

Hermes는 React Native의 디폴트 JS 엔진으로, 이를 위한 디버깅 지원을 추가했다:
- 실행 성능 분석
- 메모리 사용량 시각화
- Hermes 전용 런타임 내부 상태 확인

Hermes Engine과의 통신은 CDP 프로토콜 확장을 활용하여 구현되었으며, 이를 통해 최신 React Native 앱에서도 효과적으로 디버그할 수 있었다.

### 네이티브 모듈 탐색 및 디버깅

React Native 앱은 네이티브 모듈과 연동되는 경우가 많다. 이를 위해:
- 네이티브 모듈의 호출 트래킹
- 자바스크립트 ↔ 네이티브 데이터 흐름 시각화
- 네이티브 코드의 퍼포먼스 메트릭 수집
등의 기능을 DevTools에 포함했다.

### AsyncStorage 및 MMKV 지원 강화

React Native에서 자주 사용되는 스토리지인 AsyncStorage와 MMKV를 DevTools에서 직접 디버깅할 수 있도록 보드 UI를 개선했다. 이러한 개선은 특히 복잡한 상태 관리 로직을 디버깅하는 데 유용했다.

## 남은 과제

React Native 통합 작업은 대부분의 주요 목표를 달성했지만, 다음과 같은 기술적 부채가 남아 있다:
1. **iOS 네이티브 모듈의 제한적 지원:**
   - macOS 환경에서 풀 디버깅을 지원하지 못한다는 점.
2. **Hermes 외 엔진 지원:**
   - JSC(JavaScriptCore)에 대한 지원이 제한적이다.
3. **UI 성능 최적화:**
   - 대규모 데이터를 다룰 때 UI가 느려지는 문제가 있다.

이러한 과제들을 해결하기 위해 지속적으로 업데이트를 계획 중이다.

:::info 1줄 요약
React Native DevTools의 통합 작업을 통해 Hermes 디버깅, 네이티브 모듈 트래킹, 스토리지 관리 도구를 추가 구현하였다. 남은 과제는 JSC 지원, iOS 개발 환경 개선 등이 있다.
:::
