# 2. 기술 스택과 설계 철학

## Zig 0.15.2

메인 언어로 Zig를 선택한 이유:

- **C/C++ 수준의 성능** — SIMD `@Vector`, 수동 메모리 관리, 제로 런타임 오버헤드
- **comptime(컴파일 타임 실행)** — JS/JSX/TS/TSX/Flow 파서를 컴파일 타임에 각각 생성. 런타임 분기가 전혀 없다
- **C/C++ 직접 링크 가능** — `@cImport`로 Hermes C++ 파서를 직접 링크하는 선택지가 열려 있음 (Flow 지원 방식 B)
- **WASM 빌드 네이티브 지원** — 별도 툴체인 없이 `zig build -Dtarget=wasm32-wasi`

> "난 OXC보다 빨랐으면 좋겠는데 너무 양보하면 안되는데" — 3/18, 성능 목표

처음에는 Zig 0.14.0으로 시작했고, 3월 21일에 0.15.2로 업그레이드:
> "지그 버전업 진행해주세요 최신버전으로" — 3/21

## 메모리 전략: Arena + Index 기반 AST (D004)

```zig
// Phase-based arena allocator
// AST 노드는 포인터 대신 인덱스로 참조 → use-after-free 방지
const ast = try arena.create(Ast);
const node_idx: NodeIndex = ast.addNode(tag, main_token, data);
```

- **Arena allocator**: 파싱 → 트랜스폼 → 코드젠 단계별로 메모리를 한 번에 해제
- **24바이트 고정 노드**: 모든 AST 노드가 정확히 24바이트 — 캐시 라인에 최적화
- **인덱스 기반 참조**: 포인터 대신 `NodeIndex`(u32)로 노드를 참조. use-after-free 원천 차단

> "24, 32로 갔을때 차이점 및 성능 트레이드오프 다 말해줘" — 3/18
> "근데 bun에는 지나?" — Bun도 24바이트
> "24바이트로 가시죠" — 최종 결정

이 결정은 프로젝트 전체에서 **한 번도 변경하지 않았다**. 3/22에 `extra_data` vs 노드 플래그 논쟁(40개+ 메시지)이 있었지만, 24바이트 노드 자체는 흔들리지 않았다.

> "하면 좋은건 하자" — 선제적 개선

## 렉서: SIMD 가속 (3/25 추가)

```zig
// @Vector 16바이트 SIMD로 공백 스킵
const chunk: @Vector(16, u8) = source[pos..][0..16].*;
const spaces = chunk == @as(@Vector(16, u8), @splat(@as(u8, ' ')));
const tabs = chunk == @as(@Vector(16, u8), @splat(@as(u8, '\t')));
const mask = spaces | tabs;
```

렉서는 약 208개의 토큰 종류를 u8 플랫 enum으로 관리. SIMD는 3/25에 적합성 작업을 마무리한 후 성능 최적화 단계에서 추가됐다.

```
361057f feat(lexer): SIMD 공백 스킵 + 식별자 스캔 (@Vector 16바이트)
```

## 파서: comptime 제네릭

```zig
pub fn Parser(comptime mode: Mode) type {
    return struct {
        // mode == .ts 일 때만 타입 어노테이션 파싱 코드가 포함됨
        // mode == .jsx 일 때만 JSX 파싱 코드가 포함됨
    };
}
```

Zig의 `comptime`은 이 프로젝트에서 가장 빛났던 기능. RegExp 파서도 동일한 패턴:

```zig
fn RegExpParser(comptime emit_ast: bool) type {
    // emit_ast=false: 검증만 (현재)
    // emit_ast=true: AST 빌드 (Phase 6에서 활성화 예정)
}
```

> "C로 가자 이런건 좀 크더라도 유지보수나 안정성을 생각해야해" — 3/19, RegExp 풀 파서 선택

## Context: packed struct(u8)

파서의 "현재 상태"를 1바이트로 추적:

```zig
const Context = packed struct(u8) {
    allow_in: bool,
    in_generator: bool,
    in_async: bool,
    in_function: bool,
    is_top_level: bool,
    in_decorator: bool,
    in_ambient: bool,
    disallow_conditional_types: bool,
};
```

함수 경계에서 save/restore가 단일 u8 복사로 끝난다. 이 리팩터링은 3/19에 수행:

> "Context bitflags 리팩터링 시작해줘" — 3/19 세 번째 세션
